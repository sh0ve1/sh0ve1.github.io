HTB-OnlyForYou

OnlyForYou

Introduce

OS : Linux
Rate : Medium

该靶机涉及知识点有：LFI，subprocess.run(shell=True)的漏洞，端口转发，cypher注入，pip download导致命令执行漏洞。

Recon

nmap

  • sudo nmap –min-rate 10000 -p- -oN nmap/ports 10.10.11.210
  • sudo nmap -sT -sV -O -sC -p22,80 -oN nmap/detail 10.10.11.210
  • sudo nmap –script=vuln -p22,80 -oN nmap/vuln 10.10.11.210

vhost

  • gobuster vhost -u only4you.htb -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-20000.txt -t 30 --append-domain --follow-redirect

HTTP

  • 在/etc/hosts中添加 10.10.11.210 only4you.htb 和 10.10.11.210 beta.only4you.htb
  • 访问 only4you.htb

  • 访问 beta.only4you.htb

  • 访问http://beta.only4you.htb/source 得到名为Beta的源码，疑似该网站源码。

  • resize功能流程：

  • convert功能流程：

www-data

  • /download 处存在LFI漏洞。send_file函数

  • 尝试使用绝对路径下载 /etc/passwd，获取成功，存在任意文件读取

nginx配置文件默认地址

  • 盲猜网站源码路径（缺少开发经验，问了一下chatgpt），成功读取源码，验证成功。

  • 类似的方法猜测 only4you.htb 的源码地址。POST传参img=/var/www/only4you.htb/app.py 获取 only4you.htb 源码

  • 根据from form import sendmessage 猜测还有一个 form.py
  • form.py中 result = run([f"dig txt {domain}"], shell=True, stdout=PIPE) 存在RCE漏洞。
  • 本地开启http服务，并且创建bash反射型shell脚本。监听端口，发送payload。

User Privilege

  • 优化一下shell

  • 查看目标机器的开放端口 netstat -tnlp，其结果详解。其中有一些端口只能目标机器访问，如53，3000，8001，3306，33060，7687，7774。

  • 分析一下端口。分析发现，最吸引人的是3000，8001和7774端口
  • 先使用chisel将目标机器上的8001端口转发到kali上的9101端口

  • kalis上访问localhost:9101

  • 提交表单，抓包，发现gunicorn/20.0.4

  • gunicorn 20.0.4 存在请求走私漏洞，折腾半天，发现走不通。

  • admin/admin 弱口令登陆成功，发现跟neo4j数据库有牵扯。

  • /search 存在neo4j数据库查询功能。经测试，存在 cypher 注入，盲猜后台逻辑大致为 MATCH (o:employee) WHERE o.Nmae='*{input}*' return o

  • 利用如下payload 获取neo4j数据库中的信息，然后通过网络访问的方式传给kali。最终得到用户口令信息 john:ThisIs4You 和 admin:admin

  • 以john的身份进行 ssh 登陆，成功获取 user 权限。

Root Privilege

  • 使用 sudo -l 查看哪些指令可以以 root 权限运行。发现使用pip3 下载 http://127.0.0.1:3000 网站中的 tar.gz 压缩包时，能以root权限运行。

  • google 发现，通过 pip 下载恶意 Python 包能够导致任意代码执行。所以现在需要制作一个恶意的Python包，然后放在 http://127.0.0.1:3000 网站中。

  • 在this_is_fine_wuzzi的基础上制作恶意包。主要是修改 setup.py 中的 RunCommand 函数。

  • 先使用chisel将目标机器上的3000端口转发到kali上的9102端口，查看一下该网站具体干啥的。

  • 访问 localhost:9102 发现是一个提供 Git 服务的网站，那么通过 git 上传恶意的 Python 包即可。

  • 使用john的口令登陆

  • 在网站上创建一个仓库 eval

  • 按照网站提示，在john用户家目录中创建一个本地git仓库，存放制作的恶意Python包

  • 上传到 http://127.0.0.1:3000 中的eval仓库中

  • 执行 sudo pip3 download http://127.0.0.1:3000/*.tar.gz，成功给 /bin/bash 添加特殊权限，使用 bash -p 成功获取 root 权限

Surmary

  • 第一次接触到Neo4j，感觉功能很强大，对它的内置函数还需要深入了解一番；
  • 获取shell之后不仅要 sudo -l 还需要使用 netstat -tnlp 查看目标开放端口 和 cat /etc/services 查看服务；
  • chisel进行端口转发很方便。
